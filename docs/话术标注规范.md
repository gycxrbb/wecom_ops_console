# 话术标注规范

> 适用范围：数据标注同学、运营管理员
> 更新日期：2026-05-12

---

## 一、话术结构说明

每条话术由以下部分组成：

| 字段 | 说明 | 是否必填 |
|------|------|:--------:|
| 标题 | 话术的简短描述 | 是 |
| 内容 | 话术正文，支持占位符 | 是 |
| 分类 | 三级分类（大类 > 子类 > 细项） | 建议填 |
| RAG 摘要 | 用于语义检索的摘要 | 建议填 |
| 客户目标 | 话术针对的客户目标类型 | 建议填 |
| 干预场景 | 话术适用的干预场景 | 建议填 |
| 问题类型 | 话术应对的问题类型 | 建议填 |
| 安全级别 | 话术内容的敏感程度 | 建议填 |
| 自定义标签 | 补充关键词 | 可选 |

## 二、RAG 标签使用规则

### 2.1 客户目标 / 干预场景 / 问题类型

这三个字段是**正式检索维度**，只能从下拉列表中选择。下拉列表来自后端词表，不是随便填的。

**正确操作**：从下拉列表中选择已有标签。

**遇到新标签时**：
1. 点击下拉框右侧的"管理"按钮
2. 在弹出的标签管理面板中点击"+ 新增标签"
3. 填写 code（英文标识，如 `postpartum_weight`）和中文名（如 `产后体重管理`）
4. 创建后回到下拉框，选中新标签

**注意**：直接在输入框输入新词不会生效。系统会在保存时丢弃不在词表中的值。

### 2.2 安全级别

| 值 | 含义 | 约束 |
|----|------|------|
| general | 普通内容 | 无限制 |
| nutrition_education | 营养教育内容 | 无限制 |
| medical_sensitive | 医疗敏感内容 | 不可设为"客户可见" |
| doctor_review | 需医生审核内容 | 不可设为"客户可见" |
| contraindicated | 禁忌内容 | **禁止进入 RAG 索引** |

### 2.3 可见性

| 值 | 含义 |
|----|------|
| coach_internal | 仅教练内部使用（默认） |
| customer_visible | 可对客户可见 |

### 2.4 自定义标签

自定义标签是**弱标签**，用于补充关键词，不作为正式检索维度。

适合放的场景：
- 临时性的关键词标记，如"外卖"、"出差"
- 话术风格标记，如"高能量"、"幽默"

不适合放的场景：
- 需要反复检索过滤的业务类型 → 应新增到正式词表
- CSV 中会反复出现的问题类型 → 应新增到正式词表

## 三、分类体系说明

### 3.1 三级分类结构

```
大类（L1） > 子类（L2） > 细项（L3）
如：健康管理类 > 饮食管理 > 食物选择建议
```

每个分类有一个稳定标识（code），格式为点分层级：

```
health.diet.food_choice
community.points.earning
service.user.problem
```

### 3.2 新建分类

管理员可在页面左侧分类树上新建分类：

1. 鼠标悬停在分类节点上，点击"+"按钮
2. 填写名称和标识 code
3. 标识 code 可手动指定（如 `health.diet.new_item`），也可留空自动生成
4. 自动生成规则：父分类 code + 拼音（如"新饮食法" → `health.diet.xin_yin_shi_fa`）

**建议**：新建分类时手动指定有语义的 code，方便后续 CSV 导入引用。

## 四、CSV 导入规范

### 4.1 CSV 字段说明

| 列名 | 必填 | 说明 |
|------|:----:|------|
| title | 是 | 话术标题，最长 128 字 |
| clean_content 或 content | 是 | 话术正文 |
| category_code | 否 | 分类标识，如 `health.diet.food_choice` |
| style | 否 | 风格：professional / encouraging / competitive |
| customer_goal | 否 | 客户目标，多值用 `\|` 分隔 |
| intervention_scene | 否 | 干预场景，多值用 `\|` 分隔 |
| question_type | 否 | 问题类型，多值用 `\|` 分隔 |
| safety_level | 否 | 安全级别 |
| visibility | 否 | 可见性 |
| summary | 否 | RAG 摘要 |
| tags | 否 | 自定义标签，多值用 `\|` 分隔 |
| usage_note | 否 | 使用说明 |
| status | 否 | 状态，默认 approved |

### 4.2 category_code 的填写规则

**推荐**：始终填写 `category_code`，这是最稳定的归类方式。

```
category_code,title,clean_content,customer_goal,safety_level
health.diet.food_choice,饮食选择建议,建议选择低GI食物...,nutrition_education,general
community.points.earning,积分获取提示,本周积分翻倍...,general,
```

**不填 category_code 时**：系统会根据 `intervention_scene` / `question_type` 推导场景，再用预设映射归类。但这条路径依赖中文名，不如 code 稳定。

### 4.3 标签字段的填写规则

```
customer_goal 的值必须是词表中存在的 code
```

可以在 CSV 中填中文或 code，系统会自动解析：
- `weight_loss` ✅ 直接命中
- `体重管理` ✅ 通过别名命中
- `产后减重` ❌ 如果不在词表中，会被丢弃并报告 warning

### 4.4 导入前先 dry-run

建议每次导入前勾选"dry-run"（如果可用）或先导入少量数据验证。

导入结果中会显示：
- **新增 / 更新 / 跳过**的数量
- **错误**：必须修正的问题（如 category_code 不存在、safety_level 为 contraindicated）
- **提示**：非阻断的标签问题（如未知 customer_goal 值被忽略）

看到提示后，应先去词表中补全缺失的标签，再重新导入。

### 4.5 常见错误

| 错误信息 | 原因 | 解决方法 |
|----------|------|----------|
| 第 N 行 category_code 'xxx' 未找到对应分类 | code 不存在 | 先在页面新建该分类，或修正 code |
| 第 N 行 safety_level=contraindicated，禁止进入 RAG 索引 | 禁忌内容不允许入库 | 移除该条或降低安全级别 |
| 第 N 行 safety_level=medical_sensitive 不可设为 customer_visible | 敏感内容不可对客户可见 | 改 visibility 为 coach_internal |
| customer_goal: xxx 未命中词表 | 标签不在正式词表中 | 先通过"管理"按钮新增标签 |

## 五、标注工作流程

### 5.1 单条标注

```
1. 在左侧分类树中找到目标分类
2. 点击"新建话术"（scene_key 会自动生成）
3. 填写标题和内容
4. 展开 RAG 配置区域，选择客户目标、干预场景、问题类型
5. 如果需要的标签不在下拉列表中 → 点"管理"新增
6. 点击"保存全部"
```

### 5.2 批量标注（CSV）

```
1. 准备 CSV 文件，确保 category_code 列填写正确的分类标识
2. 确保标签字段值都在词表中（不在的先通过"管理"按钮新增）
3. 上传 CSV
4. 检查导入结果中的错误和提示
5. 有错误则修正 CSV 后重新导入
6. 有提示则补充词表后重新导入
```

### 5.3 标签新增流程

```
1. 编辑话术时发现需要新标签
2. 点击下拉框旁"管理"按钮
3. 点击"+ 新增标签"
4. 填写：
   - code：英文蛇形，如 postpartum_weight
   - 名称：中文，如 产后体重管理
5. 创建后，下拉框自动刷新
6. 选中新建的标签，保存话术
```

## 六、注意事项

1. **不要直接输入未知标签**：虽然输入框允许输入任意文字，但不在词表中的值会被丢弃。请先通过"管理"按钮新增。
2. **category_code 是稳定标识**：CSV 导入时优先使用 code，不要依赖中文名。
3. **安全级别不能乱填**：`contraindicated` 会被直接拦截，`medical_sensitive` 和 `doctor_review` 不能设为客户可见。
4. **修改分类 code 要谨慎**：改了 code 后，之前引用旧 code 的 CSV 行会报错。一般不改已有 code。
5. **定期清理词表**：如果发现重复或无用的标签，联系管理员禁用或合并。
