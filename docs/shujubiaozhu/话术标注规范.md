# 话术 RAG 标注规范 v1

## 这份文档是做什么的？

教练在跟客户沟通时，有很多经过审核的标准话术（比如饮食指导、血糖科普、情绪安抚等）。这些话术如果只是存在库里不去用，就浪费了。

**RAG（检索增强生成）** 的作用是：当教练在和某个客户对话时，系统会根据客户当前的情况，自动从话术库里找出最相关的话术，作为参考推荐给教练或 AI 助手。

**你做的事情**：给每条话术打上标签（场景、目标、安全等级等），让系统能精准地找到它。

---

## CSV 表头（复制这一行即可）

```
title,clean_content,summary,status,customer_goal,intervention_scene,content_kind,visibility,safety_level,question_type,tags,usage_note,tone,reviewer,review_note
```

---

## 每个字段怎么填

### 必填字段

| 字段 | 通俗解释 | 怎么填 | 例子 |
|------|----------|--------|------|
| **title** | 话术的名字，一眼看出这条话术是干嘛的 | 自己起一个简短标题 | 减脂控糖主食分级选择指导话术 |
| **clean_content** | 话术正文，就是实际要说的内容 | 从话术库复制过来，不要带客户隐私和二维码 | 主食可以按优先级分级选择：1. 优先优选... |
| **summary** | 这条话术适合在什么场景用，50-120 字 | 用自己的话说清楚"什么时候用、给谁用" | 外食出差专属落地话术，解决无法自制三餐的饮食困惑 |
| **status** | 这条话术审核通过了没有 | 填 `approved`（已审核）或 `active`（已上线） | approved |

### 标签字段（核心工作）

这些标签决定了系统能不能精准找到这条话术。**必须用英文 code**，对照下面的词表填写。

| 字段 | 通俗解释 | 怎么填 |
|------|----------|--------|
| **customer_goal** | 这条话术适合哪类目标的客户？可以多选 | 从下方"客户目标词表"选，多值用 `\|` 分隔 |
| **intervention_scene** | 这条话术用在什么场景？可以多选 | 从下方"场景词表"选，多值用 `\|` 分隔 |
| **question_type** | 客户问了什么问题才适合推这条话术？可以多选 | 从下方"问题类型词表"选，多值用 `\|` 分隔 |
| **safety_level** | 这条话术涉及的内容有多敏感？ | 从下方"安全等级词表"选一个 |
| **visibility** | 这条话术能不能发给客户看？ | 从下方"可见范围词表"选一个 |

### 语义增强字段（让系统推荐更精准）

| 字段 | 通俗解释 | 怎么填 |
|------|----------|--------|
| **tags** | 自由标签，给话术打上关键词 | 用 `\|` 分隔，比如 `主食\|分级\|减脂` |
| **usage_note** | 什么情况下用这条话术 | 比如"客户问主食吃什么时发送，适合餐评和习惯教育场景" |

### 可选字段（有就填，没有可以空着）

| 字段 | 通俗解释 |
|------|----------|
| **content_kind** | 话术固定填 `script`，一般不用改 |
| **tone** | 这条话术的语气风格：专业温和 / 温柔安抚 / 实用落地 / 鼓励共情 / 客观专业 |
| **reviewer** | 谁审核的 |
| **review_note** | 审核备注（比如"可直接发客户，注意结合客户实际情况调整"） |

## 词表（填标签时照着选）

**所有标签必须用英文 code，不要填中文。** 中文只是帮你理解含义的。

### 客户目标（customer_goal）

选一个或多个，用 `|` 分隔，比如 `weight_loss|glucose_control`。

| 填这个英文 code | 对应中文 | 什么时候选 |
|:---|:---|:---|
| `weight_loss` | 减脂 | 客户在减脂减重 |
| `glucose_control` | 控糖 | 客户在控糖 |
| `habit_building` | 习惯养成 | 建立饮食/运动习惯 |
| `nutrition_education` | 饮食营养教育 | 科普营养知识 |
| `exercise_adherence` | 运动坚持 | 提升运动依从性 |
| `emotion_support` | 情绪支持 | 情绪管理、心理支持 |
| `device_usage` | 设备使用 | 教客户用健康设备 |
| `maintenance` | 长期维护 | 维持期的客户 |

### 场景（intervention_scene）

选一个或多个，用 `|` 分隔，比如 `meal_review|qa_support`。

| 填这个英文 code | 对应中文 | 什么时候选 |
|:---|:---|:---|
| `meal_checkin` | 晒餐/打卡 | 客户发了餐食照片 |
| `meal_review` | 餐评 | 给客户的餐食做点评 |
| `obstacle_breaking` | 破阻力 | 客户遇到困难需要突破 |
| `habit_education` | 习惯教育 | 教客户建立好习惯 |
| `emotional_support` | 情绪支持 | 客户情绪低落或焦虑 |
| `qa_support` | 问题答疑 | 客户问了问题 |
| `period_review` | 周期复盘 | 做阶段性回顾 |
| `maintenance` | 长期维护 | 维持期日常跟进 |
| `abnormal_intervention` | 异常干预 | 客户指标异常需要关注 |
| `data_monitoring` | 数据监测 | 看客户的数据变化 |
| `device_guidance` | 设备指导 | 指导客户使用设备 |
| `points_operation` | 积分/社群运营 | 群内活动、积分激励 |

### 问题类型（question_type）

选一个或多个，用 `|` 分隔。如果话术不属于某个具体问题，可以不填。

| 填这个英文 code | 对应中文 | 什么时候选 |
|:---|:---|:---|
| `dining_out` | 外食/外卖 | 客户出差、在外面吃 |
| `carb_choose` | 主食选择 | 客户问吃什么主食 |
| `low_calorie` | 低卡食材 | 客户问有没有低热量食物 |
| `late_night_snack` | 晚间零食 | 客户晚上想吃东西 |
| `craving` | 嘴馋/食欲 | 客户说想吃不能吃的 |
| `hunger` | 饥饿感 | 客户说总是饿 |
| `food_safety` | 食材安全 | 客户问某种食物能不能吃 |
| `high_glucose` | 血糖偏高 | 客户血糖测出来高了 |
| `blood_fluctuation` | 血糖波动 | 客户血糖忽高忽低 |
| `data_monitoring` | 数据解读 | 客户的数据需要分析 |
| `no_checkin` | 不打卡 | 客户不积极打卡 |
| `low_motivation` | 动力不足 | 客户想放弃、没动力 |
| `device_usage` | 设备使用 | 客户不会用设备 |
| `plateau` | 平台期 | 客户体重/指标停滞不前 |

### 安全等级（safety_level）

**只能选一个**。拿不准的宁可标高一级。

| 填这个英文 code | 对应中文 | 什么意思 |
|:---|:---|:---|
| `general` | 通用 | 普通饮食/习惯话术，没有敏感内容 |
| `nutrition_education` | 营养科普 | 营养知识科普，不能承诺疗效 |
| `medical_sensitive` | 医疗敏感 | 涉及血糖/血压等指标解读，**只能内部参考，不能直接发给客户** |
| `doctor_review` | 需医生确认 | 涉及用药/诊疗建议，**必须医生看过才能用** |
| `contraindicated` | 禁用 | 绝对不能发给客户（这条话术不会出现在系统推荐里） |

### 可见范围（visibility）

**只能选一个**。

| 填这个英文 code | 什么意思 |
|:---|:---|
| `coach_internal` | 仅教练内部参考，不能直接发给客户 |
| `customer_visible` | 可以发给客户看（但仍需教练自己确认后再发） |

> 规则：安全等级为 `medical_sensitive` 或 `doctor_review` 的话术，可见范围**必须是** `coach_internal`。

## 一个完整的例子

下面是 CSV 里一行的实际内容（用逗号分隔），对照着看每个字段怎么填：

```
出差外食外卖场景点餐指导话术,出差和在外用餐，可以参考这套简单好执行的选择方式：便利店优先选...,外食、出差、外卖专属落地话术，解决无法自制三餐的饮食困惑，可直接复制发给客户。,approved,weight_loss|glucose_control,meal_review|qa_support,script,customer_visible,nutrition_education,dining_out,外食|外卖|出差,客户出差或外卖点餐时发送,实用落地,主管,可直接发客户，适合出差和外卖场景
```

逐个字段解读：
- `出差外食外卖场景点餐指导话术` — **title**
- `出差和在外用餐...` — **clean_content**（话术正文）
- `外食、出差、外卖专属...` — **summary**
- `approved` — **status**
- `weight_loss|glucose_control` — **customer_goal**（两个目标，用 `|` 分隔）
- `meal_review|qa_support` — **intervention_scene**
- `script` — content_kind（话术固定填 `script`）
- `customer_visible` — **visibility**（可以发给客户）
- `nutrition_education` — **safety_level**（营养科普）
- `dining_out` — **question_type**
- `外食|外卖|出差` — **tags**（自由标签）
- `客户出差或外卖点餐时发送` — **usage_note**（使用场景说明）
- `实用落地` — **tone**
- `主管` — reviewer
- `可直接发客户...` — review_note

## 标注检查清单

交之前对照这个清单过一遍：

- [ ] title 简短明确，一眼能看出话术主题
- [ ] clean_content 里没有客户姓名、电话、二维码等隐私信息
- [ ] summary 说清楚了"什么时候用、给谁用"（50-120 字）
- [ ] content_kind 填了 `script`
- [ ] customer_goal 至少选了一个
- [ ] intervention_scene 至少选了一个
- [ ] safety_level 选了，拿不准的标 `medical_sensitive`
- [ ] safety_level 为 medical_sensitive 或 doctor_review 的话术，visibility 是 coach_internal
- [ ] tags 填了关键词标签
- [ ] usage_note 填了使用场景说明
- [ ] tone 选了语气风格
- [ ] 所有标签用的是英文 code，不是中文
