# 素材库 RAG 标注规范 v1

## 这份文档是做什么的？

系统里有很多素材（图片、视频、PDF 等），教练在跟客户沟通时可能会用到。但如果素材只是堆在库里没有分类，需要的时候就找不到。

**RAG（检索增强生成）** 的作用是：当教练在和某个客户对话时，系统会根据客户当前的情况，自动推荐最相关的素材（比如一张血糖标准参考图、一个设备使用教程视频）。

**你做的事情**：给每个素材打上标签（类型、场景、目标、安全等级等），并写好文字描述，让系统能精准地找到它。

---

## CSV 表头（复制这一行即可）

```
source_type,title,summary,content_kind,status,rag_enabled,visibility,safety_level,copyright_status,customer_sendable,alt_text,usage_note,customer_goal,intervention_scene,question_type,tags,material_ref,file_name,file_hash,transcript,public_url
```

---

## 每个字段怎么填

### 必填字段（缺一不可，否则整行跳过）

| 字段 | 通俗解释 | 怎么填 | 例子 |
|------|----------|--------|------|
| **title** | 素材的名字 | 给素材起一个简短标题 | 血糖标准参考图 |
| **summary** | 一句话描述这个素材是什么，50-120 字 | 说清楚内容是什么、能给谁看 | 展示空腹和餐后血糖的正常范围及糖尿病诊断标准 |
| **status** | 这个素材审核过了没有 | 填 `approved` 或 `active` | approved |
| **rag_enabled** | 是否让系统检索这个素材 | 填 `yes` | yes |
| **content_kind** | 素材是什么类型 | 从下方"素材类型词表"选一个 | image |
| **visibility** | 这个素材能不能发给客户看 | 从下方"可见范围词表"选一个 | coach_internal |
| **safety_level** | 内容有多敏感 | 从下方"安全等级词表"选一个 | general |
| **copyright_status** | 版权情况 | `owned`（自有）/ `licensed`（已授权）/ `unknown`（未知，会跳过） | owned |

`source_type` 列固定填 `material`，不填也行（系统默认当素材处理）。

### 素材匹配（三选一，必须能匹配到系统里的素材）

每行必须能对应到系统素材库里已有的素材，否则这行会被跳过。

| 字段 | 通俗解释 | 最靠谱的填法 |
|------|----------|--------------|
| **material_ref** | 直接写素材 ID | `materials.id=12`（去素材库里找 ID） |
| **file_hash** | 按文件指纹匹配 | 从素材库复制 hash 值 |
| **file_name** | 按文件名匹配 | 写原始文件名，比如 `雅培血糖仪的价值.mp4` |

匹配优先级：material_ref > file_hash > file_name。**建议都用 material_ref，最稳定。**

### 语义标注字段（核心工作）

这些字段越完整，系统推荐越精准。质量等级会自动计算：

| 字段 | 通俗解释 | 怎么填 |
|------|----------|--------|
| **alt_text** | 图片/表情包的文字描述（图片和表情包必填） | 用文字描述图片内容，比如"空腹血糖3.9-6.1为正常..." |
| **transcript** | 视频的文字内容（和 alt_text 二选一） | 把视频里说了什么写下来 |
| **usage_note** | 什么时候用这个素材 | 比如"血糖异常时向客户解释标准范围" |
| **customer_goal** | 适合哪类目标的客户？可以多选 | 从"客户目标词表"选，多值用 `\|` 分隔 |
| **intervention_scene** | 用在什么场景？可以多选 | 从"场景词表"选，多值用 `\|` 分隔 |
| **question_type** | 客户遇到什么问题时用？可以多选 | 从"问题类型词表"选，多值用 `\|` 分隔 |
| **tags** | 自由标签 | 用 `\|` 分隔，比如 `血糖\|标准\|参考图` |

#### 质量等级（自动计算）

| 等级 | 条件 | 能不能被系统推荐 |
|:---:|------|:---:|
| **ok** | title + summary + (alt_text 或 transcript) + usage_note + tags 都有 | ✅ 能 |
| **medium** | 有 title + summary + usage_note + tags，缺图片说明 | ✅ 能 |
| **weak** | 只有零散信息 | ❌ 不能，**等于白标** |

> 标注目标是让每条素材都达到 **ok** 或 **medium**。

### 可选字段（有就填，没有空着）

| 字段 | 通俗解释 |
|------|----------|
| **customer_sendable** | 导入辅助字段（yes/no），填了 yes 会自动映射成 visibility=customer_visible。但如果没填 public_url 会被降回 coach_internal |
| **public_url** | 素材的公开访问链接（visibility=customer_visible 时必须有） |
| **folder_name** | 素材所在文件夹的名字，用来辅助匹配 |

## 词表（填标签时照着选）

**所有标签必须用英文 code，不要填中文。** 中文只是帮你理解含义的。

### 素材类型（content_kind）

**只能选一个。**

| 填这个英文 code | 对应中文 | 什么时候选 | 额外要求 |
|:---|:---|:---|:---|
| `image` | 图片 | 科普图、参考图、截图等 | 必须填 alt_text |
| `video` | 视频 | 教程视频、科普视频（mp4/mov 等） | 必须填 transcript 或 alt_text |
| `meme` | 表情包 | 用来活跃气氛、鼓励客户的表情包 | 必须填 alt_text + usage_note |
| `file` | 文件 | PDF、表格、文档等附件 | 必须填 summary + usage_note |

> ⚠️ 视频必须标 `video`，不要标 `file`；表情包必须标 `meme`，不要标 `image`。

### 客户目标（customer_goal）

选一个或多个，用 `|` 分隔，比如 `glucose_control|weight_loss`。

| 填这个英文 code | 对应中文 | 什么时候选 |
|:---|:---|:---|
| `weight_loss` | 减脂 | 客户在减脂减重 |
| `glucose_control` | 控糖 | 客户在控糖 |
| `habit_building` | 习惯养成 | 建立饮食/运动习惯 |
| `nutrition_education` | 饮食营养教育 | 科普营养知识 |
| `exercise_adherence` | 运动坚持 | 提升运动依从性 |
| `emotion_support` | 情绪支持 | 情绪管理、心理支持 |
| `device_usage` | 设备使用 | 教客户用健康设备 |
| `maintenance` | 长期维护 | 维持期的客户 |

### 场景（intervention_scene）

选一个或多个，用 `|` 分隔，比如 `meal_review|qa_support`。

| 填这个英文 code | 对应中文 | 什么时候选 |
|:---|:---|:---|
| `meal_checkin` | 晒餐/打卡 | 客户发了餐食照片 |
| `meal_review` | 餐评 | 给客户的餐食做点评 |
| `obstacle_breaking` | 破阻力 | 客户遇到困难需要突破 |
| `habit_education` | 习惯教育 | 教客户建立好习惯 |
| `emotional_support` | 情绪支持 | 客户情绪低落或焦虑 |
| `qa_support` | 问题答疑 | 客户问了问题 |
| `period_review` | 周期复盘 | 做阶段性回顾 |
| `maintenance` | 长期维护 | 维持期日常跟进 |
| `abnormal_intervention` | 异常干预 | 客户指标异常需要关注 |
| `data_monitoring` | 数据监测 | 看客户的数据变化 |
| `device_guidance` | 设备指导 | 指导客户使用设备 |
| `points_operation` | 积分/社群运营 | 群内活动、积分激励 |

### 问题类型（question_type）

选一个或多个，用 `|` 分隔。如果素材不属于某个具体问题，可以不填。

| 填这个英文 code | 对应中文 | 什么时候选 |
|:---|:---|:---|
| `dining_out` | 外食/外卖 | 客户出差、在外面吃 |
| `carb_choose` | 主食选择 | 客户问吃什么主食 |
| `low_calorie` | 低卡食材 | 客户问有没有低热量食物 |
| `late_night_snack` | 晚间零食 | 客户晚上想吃东西 |
| `craving` | 嘴馋/食欲 | 客户说想吃不能吃的 |
| `hunger` | 饥饿感 | 客户说总是饿 |
| `food_safety` | 食材安全 | 客户问某种食物能不能吃 |
| `high_glucose` | 血糖偏高 | 客户血糖测出来高了 |
| `blood_fluctuation` | 血糖波动 | 客户血糖忽高忽低 |
| `data_monitoring` | 数据解读 | 客户的数据需要分析 |
| `no_checkin` | 不打卡 | 客户不积极打卡 |
| `low_motivation` | 动力不足 | 客户想放弃、没动力 |
| `device_usage` | 设备使用 | 客户不会用设备 |
| `plateau` | 平台期 | 客户体重/指标停滞不前 |

### 可见范围（visibility）

**只能选一个。**

| 填这个英文 code | 什么意思 |
|:---|:---|
| `coach_internal` | 仅教练内部参考，不能直接发给客户 |
| `customer_visible` | 可以发给客户看（但仍需教练自己确认后再发） |

### 安全等级（safety_level）

**只能选一个。** 拿不准的宁可标高一级。

| 填这个英文 code | 对应中文 | 什么意思 |
|:---|:---|:---|
| `general` | 通用 | 普通素材，没有敏感内容 |
| `nutrition_education` | 营养科普 | 营养知识科普，不能承诺疗效 |
| `medical_sensitive` | 医疗敏感 | 涉及血糖/血压等指标，**只能内部参考，不能直接发给客户** |
| `doctor_review` | 需医生确认 | 涉及用药/诊疗，**必须医生看过才能用** |
| `contraindicated` | 禁用 | 绝对不能发给客户（不会出现在系统推荐里） |

> ⚠️ **联动规则**：安全等级为 `medical_sensitive` 或 `doctor_review` 的素材，可见范围**必须是** `coach_internal`。

## 一个完整的例子

下面是 CSV 里一行的实际内容（用逗号分隔）：

```
material,血糖标准参考图,展示空腹和餐后血糖的正常范围及糖尿病诊断标准,image,approved,yes,coach_internal,general,owned,no,空腹血糖3.9-6.1为正常 餐后2小时低于7.8为正常 用于判断血糖控制情况,血糖异常时向客户解释标准范围,glucose_control,abnormal_intervention,data_monitoring,血糖|标准|参考图,materials.id=12,,
```

逐个字段解读：
- `material` — source_type，固定填 material
- `血糖标准参考图` — title
- `展示空腹和餐后...` — summary
- `image` — content_kind
- `approved` — status
- `yes` — rag_enabled
- `coach_internal` — visibility
- `general` — safety_level
- `owned` — copyright_status
- `no` — customer_sendable
- `空腹血糖3.9-6.1...` — alt_text
- `血糖异常时向客户解释...` — usage_note
- `glucose_control` — customer_goal
- `abnormal_intervention` — intervention_scene
- `data_monitoring` — question_type
- `血糖|标准|参考图` — tags
- `materials.id=12` — material_ref
- 末尾两个逗号 — file_name 和 file_hash 为空

## 标注检查清单

交之前对照这个清单过一遍：

- [ ] 每行都能匹配到素材库里的一个素材（material_ref / file_hash / file_name）
- [ ] title 简短明确
- [ ] summary 说清楚了内容是什么（50-120 字）
- [ ] content_kind 选对了：视频标 `video`，表情包标 `meme`，图片标 `image`
- [ ] 图片和表情包都填了 alt_text
- [ ] 视频填了 transcript 或 alt_text
- [ ] customer_goal 和 intervention_scene 至少各选了一个
- [ ] safety_level 选了，拿不准的标 `medical_sensitive`
- [ ] safety_level 为 medical_sensitive 或 doctor_review 的素材，visibility 是 coach_internal
- [ ] 所有标签用的是英文 code，不是中文
- [ ] 质量等级至少是 medium（title + summary + usage_note + tags 都有）
