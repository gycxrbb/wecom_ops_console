# 积分推送"当月积分排名"实施规划

> 状态：待实施  
> 日期：2026-05-13

---

## 1. 需求概述

在积分推送（发送中心 + 自动排行推送）中新增 **"当月积分排名"** 排序方式，与现有"总积分排名"并列可选。

**"当月"的定义**：以 CRM `groups.start_time` 为起点，每 30 天为一个周期，当前落在哪个周期就用该周期计算积分。

- 例：`start_time = 2026-03-15`，现在是 2026-05-13，当前周期为 `2026-05-15 ~ 2026-06-14`（第三个周期）。但 5-13 还在上一个周期 `2026-04-15 ~ 2026-05-14`。

**排名行显示**：月积分为主要分数，备注本周 + 总积分。
- 例：`🥇 张三 580分（本周+120，总积分2345）`

### 步骤 1：`crm_points_metrics.py` — 新增周期计算

新增函数，根据 `groups.start_time` 计算当前 30 天周期：

```python
def get_group_custom_month_range(start_time: datetime) -> tuple[datetime, datetime]:
    """根据群的 start_time，计算当前所在的 30 天周期范围。
    
    例：start_time=3月15日 → 周期为 [15日, 次月14日]
    当前时间落在哪个周期，就返回该周期的起止时间。
    """
    tz = ZoneInfo('Asia/Shanghai')
    now = datetime.now(tz)
    
    period_start = start_time
    while period_start + timedelta(days=30) <= now:
        period_start += timedelta(days=30)
    
    period_end = period_start + timedelta(days=30)
    return period_start, period_end
```

## 4. 排名指标对照

| 值 | 含义 | 排序依据 | 显示格式 |
|---|---|---|---|
| `current_points` | 总积分排名 | `customers.points` | `{name} {总分}分（本周+{week}）` |
| `week_points` | 本周积分排名 | `point_logs` 本周净增 | 同上（分数显示总积分） |
| `month_points` | 本月积分排名 | `point_logs` 自然月净增 | 同上 |
| **`custom_month_points`** | **当月积分排名（新增）** | **`point_logs` 自定义30天周期净增** | **`{name} {月积分}分（本周+{week}，总积分{total}）`** |

---

## 5. 数据流（当月积分排名）

```
1. 前端选择"当月积分排名" → rank_metric='custom_month_points'
2. 后端 fetch_crm_group_members_bulk → SQL 新增 g.start_time
3. 对每个群，用 start_time 计算当前 30 天周期 [period_start, period_end)
4. fetch_customer_custom_month_points → 从 point_logs 聚合该周期的净积分
5. 按custom_month_points 排序
6. 排名行显示：月积分为主要分数 + 备注本周+总积分
7. 生成 markdown 消息 → 拆分检测 → 发送
```
