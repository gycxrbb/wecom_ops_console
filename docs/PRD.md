---

## 1. 项目背景

当前训练营/教练团队在企业微信群内进行内容推送时，主要依赖脚本、手工改配置或人工复制发送。现状存在以下问题：

1. 发送内容依赖研发/懂技术同学维护，运营和教练无法自助完成。
2. 群与 Webhook 映射缺乏统一管理，容易误发、漏发或暴露敏感配置。
3. 定时任务、模板、素材与发送记录缺乏统一平台管理，排查问题成本高。
4. 消息类型不统一，教练无法低门槛使用文本、Markdown、图文、图片、文件、模板卡片等能力。
5. 缺乏审批、预览、测试群发送、失败重试、频控等生产级能力。

因此需要建设一套面向运营与教练的可视化后台，使非技术人员可以通过页面完成群选择、模板选择、自定义编辑、立即发送、定时发送、审批发布和历史追踪。

---

## 2. 项目目标

### 2.1 业务目标

1. 将群消息发送从“开发维护”升级为“教练/运营自助操作”。
2. 提升内容推送效率与标准化程度，降低误发、错发风险。
3. 支持训练营、答疑群、复训群、VIP 群等多类群运营场景。
4. 建立可追踪、可审计、可复用、可扩展的消息运营平台。

### 2.2 产品目标

1. 支持群管理、模板管理、素材管理、发送中心、定时任务、发送记录、审批流、用户权限。
2. 支持文本、Markdown、图文、图片、文件、模板卡片、原始 JSON 发送。
3. 支持立即发送、测试群发送、定时发送、批量群发、失败重试。
4. 支持模板变量渲染、预览、跳过日期、跳过周末、审批后生效。

### 2.3 技术目标

1. 前后端分层清晰，支持后续云端部署与横向扩展。
2. Webhook 仅后端持有，前端不可见。
3. 调度、发送、日志、审批、频控可独立扩展。
4. 提供统一接口规范，降低前后端协作成本。

## 5. 核心使用场景

### 场景 1：教练发送一条即时提醒

1. 教练进入发送中心。
2. 选择目标群。
3. 选择消息类型与模板。
4. 微调内容并预览。
5. 点击“立即发送”。
6. 若命中审批规则，则进入待审批；否则直接发送。
7. 在发送记录中查看结果。

### 场景 2：教练配置训练营一周定时推送

1. 教练选择多个时间槽。
2. 绑定群、模板、主题变量。
3. 设置 Cron、跳过周末、跳过日期。
4. 提交审批。
5. 审批通过后自动生效。
6. 定时任务到点执行并生成发送记录。

### 场景 3：管理员配置新群

1. 管理员在群管理中新增群。
2. 填写群名称、标签、Webhook。
3. 指定该群为正式群或测试群。
4. 可绑定默认模板集。

### 场景 4：教练发测试群进行预览验证

1. 教练编辑内容。
2. 点击“发送到测试群”。
3. 后端自动改用测试群 Webhook。
4. 查看发送结果后再发正式群。

### 场景 5：发送失败后重试

1. 定时任务执行失败。
2. 系统记录错误信息与返回结果。
3. 管理员在发送记录中点击重试。
4. 系统重新入队并发送。

## 6. 功能需求

## 6.1 登录与认证

### 功能描述
- 支持账号密码登录。
- 支持 JWT 认证与刷新。
- 支持当前用户信息查询。
- 支持登出。

### 规则
- 用户分角色。
- 令牌过期后需要刷新。
- 重要操作必须校验权限。

---

## 6.2 群管理

### 功能描述
- 新增/编辑/停用群
- 支持群标签管理
- 支持正式群/测试群区分
- 支持默认模板集
- 支持 Webhook 加密存储

### 关键字段
- 群名称
- 展示别名
- 群类型
- 标签
- 状态
- Webhook
- 是否默认测试群

### 业务规则
- 前端不可返回 Webhook 明文。
- 停用群不可用于新建发送任务。
- 删除群前需校验是否存在生效中的任务。

---

## 6.3 模板管理

### 功能描述
- 新增/编辑/复制/禁用模板
- 支持系统模板与自定义模板
- 支持消息类型区分
- 支持模板变量说明
- 支持预览

### 支持的消息类型
- text
- markdown
- news
- image
- file
- template_card
- raw_json

### 模板字段
- 模板名称
- 分类
- 消息类型
- 正文内容
- 变量 Schema
- 默认变量值
- 适用场景
- 是否系统模板
- 状态

### 业务规则
- 系统模板仅管理员可修改。
- 教练可基于系统模板复制成私有模板。
- 模板变量渲染失败时不得发送。

---

## 6.4 素材管理

### 功能描述
- 图片上传
- 文件上传
- 素材标签
- 素材引用关系追踪

### 业务规则
- 文件大小和类型受配置限制。
- 被模板引用的素材不可直接物理删除，只能软删除。

---

## 6.5 发送中心

### 功能描述
- 选择群
- 选择模板
- 选择消息类型
- 编辑内容
- 预览内容
- 测试群发送
- 立即发送
- 提交审批
- 定时发送

### 业务规则
- 若消息类型为 template_card/raw_json，则前端提供 JSON 编辑器。
- 测试群发送不影响正式任务状态。
- 立即发送也需要写入发送记录。
- 若启用审批且当前用户非管理员，则必须先审批后执行。

---

## 6.6 定时任务管理

### 功能描述
- 创建一次性任务
- 创建 Cron 周期任务
- 启停任务
- 复制任务
- 设置跳过周末
- 设置跳过日期
- 查看下次执行时间

### 任务字段
- 任务名称
- 目标群
- 模板
- 消息类型
- 渲染参数
- 调度类型
- Cron 表达式/执行时间
- 时区
- 是否启用
- 是否需要审批
- 跳过周末
- 跳过日期

### 业务规则
- 停用任务不入调度。
- 审批未通过的任务不能激活。
- 任务修改后重新计算下次执行时间。

---

## 6.7 审批流

### 功能描述
- 教练提交发送审批
- 管理员批准/拒绝
- 审批记录可追溯
- 审批通过后自动执行或生效

### 规则
- 可配置哪些消息类型或群必须审批。
- 可配置定时任务创建必须审批。
- 拒绝后需填写原因。

---

## 6.8 发送记录与日志

### 功能描述
- 记录发送请求与响应
- 展示状态（成功/失败/待审批/已取消）
- 支持搜索与筛选
- 支持失败重试
- 支持查看渲染后内容

### 业务规则
- 所有发送行为都必须落库。
- 失败重试不能突破频控上限。

---

## 6.9 看板

### 指标示例
- 群总数
- 启用群数
- 模板总数
- 任务总数
- 今日发送数
- 今日成功率
- 待审批数

---

## 7. 非功能需求

## 7.1 性能
- 常规查询接口响应时间 P95 < 500ms
- 页面首屏可用时间 < 3s（内网环境）
- 单实例支持 1000+ 任务管理规模

## 7.2 可用性
- 发送服务失败可重试
- 调度服务异常恢复后可继续补偿扫描
- 数据写入与发送过程可追踪

## 7.3 安全
- Webhook 加密存储
- 前端不暴露敏感配置
- 所有写操作需鉴权
- 审计日志保留关键行为

## 7.4 可维护性
- 模块化设计
- 接口统一规范
- 日志结构化
- 配置中心化

---

## 8. 关键业务规则

1. Webhook 只保存在后端。
2. 消息发送必须经过统一发送服务。
3. 群、模板、任务删除以软删除为主。
4. 系统必须支持测试群验证。
5. 定时任务必须支持跳过日期与跳过周末。
6. 审批通过前，任务不得进入有效发送状态。
7. 所有消息需记录请求体、返回体、耗时与结果。
8. 失败重试必须写入新的发送尝试日志。

---

## 9. 成功指标（KPI）

1. 教练自助发送占比 > 80%
2. 模板复用率 > 60%
3. 手工误发率下降 70%
4. 失败发送定位耗时下降 80%
5. 训练营内容配置耗时下降 50%

## 11. 风险与对策

### 风险 1：Webhook 配置错误导致误发
对策：
- 后端加密存储
- 正式群/测试群分组管理
- 发送前预览
- 关键群开启审批

### 风险 2：消息类型配置复杂，教练难以上手
对策：
- 优先提供模板与预设
- JSON 类型隐藏到高级模式
- 提供测试群发送

### 风险 3：高并发定时点触发造成限流
对策：
- Redis 队列
- 单机器人维度频控
- 失败重试和延迟重投

### 风险 4：任务过多，排查复杂
对策：
- 标准化日志
- 看板 + 筛选 + 任务复制
- 明确审批和发布链路

## 12. 验收标准

### 核心功能验收
- 教练可以不改代码完成内容发送。
- 前台可以选择群、模板、消息类型并预览。
- 后端根据 group_id 自动切换 Webhook。
- 支持立即发送、测试群发送、定时任务。
- 支持审批和失败重试。
- 所有发送结果均可追踪。

### 安全验收
- 前端接口不返回 Webhook 明文。
- 非管理员无法查看敏感群配置。
- 所有操作有用户归属。

### 技术验收
- 接口文档齐全。
- 数据表结构齐全。
- 定时任务可恢复。
- 发送服务可限流。
