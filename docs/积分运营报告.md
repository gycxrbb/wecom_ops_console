# 积分运营系统 — 当前能力综合报告

> 更新日期：2026-04-20 | 覆盖范围：积分洞察、话术模板、排行生成、全局 PK、1v1 配套、运营计划

---

## 一、系统定位

面向首钢减重健康群项目，基于企业微信机器人的 **积分激励自动化运营平台**。覆盖 20+ CRM 外部社群，支持从数据采集 → 洞察识别 → 话术生成 → 定时推送 → 1v1 跟进的全链路运营闭环。

---

## 二、核心能力矩阵

### 2.1 积分洞察识别（10 种个人场景 + 2 种群场景）

系统从 CRM 数据库 `point_logs` 表自动分析用户近 14 天的行为数据，识别以下运营场景：

| # | 场景 | 识别逻辑 | 运营价值 |
|---|------|----------|----------|
| 1 | **头部领先** (TOP3) | 排名 ≤ 3 | 稳住头部用户，金百花奖冲刺区 |
| 2 | **前六冲刺** (TOP4-6) | 排名 4-6 | 第一梯队激励，金百花奖争夺 |
| 3 | **前十竞争** (TOP7-10) | 排名 7-10 | 差距不大，增加打卡频次可逆袭 |
| 4 | **连续活跃** | 14 天内活跃 ≥ 7 天 | 树立标杆，带动群氛围 |
| 5 | **积分暴涨** | 近 3 天日均 > 前 7 天日均 × 2 | 发现黑马，及时表扬 |
| 6 | **强势回归** | 近 3 天有积分，之前无记录 | 欢迎归队，重新融入 |
| 7 | **掉队归队** | 早期活跃 → 中断 3+ 天 → 最近重新活跃 | 针对性鼓励跟上节奏 |
| 8 | **进步飞快** | 近 7 天日均 > 前 7 天日均 × 1.5 | 肯定进步势头 |
| 9 | **潜水用户** | 14 天活跃 ≤ 2 天，有积分基础，排名 > 10 | 温和带动，减少沉寂 |
| 10 | **后段激励** | 排名后 6 名（群 ≥ 10 人） | 绿草莓奖反向激励 |

群级洞察：**群组领先**（群排名 TOP3）、**本周活跃群**（周积分 > 0）。

### 2.2 话术模板库（12 场景 × 3 风格 = 36 条）

基于首钢减重健康群积分激励专属话术 Excel 深度定制，每个场景支持 3 种运营风格：

| 风格 | 适用场景 | 语调特征 |
|------|----------|----------|
| **专业正式版** | 首钢国企群、严肃健康管理场景 | 规范、数据化、引导性强 |
| **温柔陪伴版** | 女性为主群、氛围温暖群 | 情感化、鼓励式、共情强 |
| **活泼接地气版** | 年轻群体、活跃群 | 网感强、幽默、有紧迫感 |

12 个场景全覆盖：top_leader、top_six、top_ten、consistent、surge、comeback、dropout_recovery、rapid_progress、lurker_remind、reverse_bottom、daily_remind、group_pk。

所有话术自动填充 `{name}` `{rank}` `{detail}` `{activity}` 占位符，无需手动编辑。

### 2.3 群内排行消息生成

**性能**：20+ 群批量生成，单次 point_logs 查询（非逐群串行），预计 10 秒内完成。

**消息内容**：
- 群名 + 群总分（含本周/本月数据）
- 成员排行榜（Top N，前三名奖牌 emoji）
- 智能亮点识别（自动匹配话术，最多展示 5 人）
- 结尾激励话术

**可配置项**：Top N 上限、排序指标（当前/本周/本月积分）、话术风格。

### 2.4 跨群全局排行（20 社群 PK）

**独立功能**：一键生成跨群总览消息，发送到运营指定群。

**消息内容**：
- 覆盖社群数 + 活跃成员数
- 全局个人 TOP10（带所属群名）
- 社群 PK 排名（按群总分排序，含人数、本周增量）
- 跨群 PK 话术（3 种风格）

### 2.5 1v1 私发消息自动生成

系统根据洞察结果自动生成 1v1 跟进动作列表，供运营人员确认后发送：

| 动作类型 | 触发条件 | 消息内容 |
|----------|----------|----------|
| 表扬 TOP3 | 排名 ≤ 3 | 鼓励稳住排名 |
| 表扬连续打卡 | 14 天活跃 ≥ 7 天 | 肯定坚持 |
| 3 天未打卡提醒 | 回归用户 | 温和提醒 |
| 2 周未打卡关怀 | 潜水用户 | 关怀+引导 |
| 鼓励冲刺前 6 | 排名 7-8 | 推一把 |
| 获奖恭喜 | 结营当天 TOP3 | 恭喜 |
| 后段鼓励 | 后 6 名 | 绿草莓奖温柔激励 |
| 表扬暴涨 | 积分暴涨 | 发现黑马 |
| 表扬进步 | 进步飞快 | 肯定势头 |

优先级逻辑：结营当天特殊处理 → 日常按 head/top_leader > surge > progress > consistent > near_top6 > comeback > lurker > reverse_bottom 排序。每群最多生成 20 条。

### 2.6 积分运营计划（6 阶段编排）

系统支持创建 `points_campaign` 模式的运营计划，自动生成 6 个阶段：

| 阶段 | 触发方式 | 核心动作 | 1v1 配套 |
|------|----------|----------|----------|
| **每日发榜** | 每天 18:00 cron | 发榜、表扬前 6、表扬连续打卡、鼓励回归、提醒潜水、社群 PK | 私发 TOP3 稳住 / 连续打卡肯定 / 3 天未打卡提醒 |
| **每周复盘** | 每周日 19:30 | 周积分总结、周 TOP6、表扬黑马、鼓励掉队、社群 PK、轻科普 | 表扬进步最快 / 邀请分享心得 / 2 周未打卡关怀 |
| **冲刺 14-7 天** | 倒计时 14~7 天 | 倒计时提醒、动员前 6、中间段冲刺、后段逆袭 | 重点用户 @ 提醒 / 差几分进前 6 私发鼓励 |
| **冲刺 7-3 天** | 倒计时 7~3 天 | 早晚双重动员、强化前 6、重点鼓励中间 | 差几分进前 6 @ / 3 天未打卡紧急提醒 |
| **结营前 1 天** | 倒数第 1 天 | 最终冲刺、最后打卡号召 | TOP10 务必打卡 / 后 6 名最后鼓励 |
| **结营当天** | 倒数第 0 天 | 公布最终榜单、金百花奖/绿草莓奖公示、感谢、社群 PK 最终排名 | 获奖恭喜 / 绿草莓奖温柔鼓励 |

**触发器类型**：
- `daily`：每天定时 → APScheduler cron
- `weekly`：每周固定日 → APScheduler cron
- `countdown_range`：倒计时范围（一次性任务）
- `final_day`：倒数第 N 天（一次性任务）
- `manual`：手动触发（仅创建草稿，不调度）

### 2.7 CRM 群绑定与发送管理

| 功能 | 说明 |
|------|------|
| 单群绑定 | CRM 群 ↔ 本地企微发送群一一映射 |
| 一键批量绑定 | 所有 CRM 群统一绑定到同一发送群 |
| 启用/禁用 | 按绑定维度开关 |
| 发送队列 | 生成的排行消息加入批量发送队列 |
| 队列定时发送 | 队列内容打包为 Schedule，支持一次性/周期定时 |

### 2.8 审批流程

- 管理员 (admin) 发布计划 → 自动通过审批
- 教练 (coach) 发布 → 进入待审批状态，管理员审批后执行
- 不需要审批 → 直接进入调度

## 三、数据流全景

```
CRM 数据库 (mfgcrmdb)
  ├── customer_groups (群成员关系)
  ├── customers (客户信息)
  └── point_logs (积分流水)
        │
        ▼
  ┌─────────────────────────────────┐
  │  crm_group_directory            │  批量加载群成员 + 周月积分
  │  crm_points_insights            │  10 种洞察场景识别（1 次 SQL）
  │  crm_points_ranking             │  逐群排行消息 + 1v1 动作生成
  │  crm_global_ranking             │  跨群全局 TOP10 + 社群 PK
  │  crm_speech_templates           │  36 条话术自动匹配填充
  │  crm_1v1_actions                │  9 类 1v1 私发动作生成
  └─────────────────────────────────┘
        │
        ▼
  发送中心 → 队列 → 定时/即时发送
  运营计划 → 6 阶段编排 → 发布 → APScheduler 自动执行
```

## 四、前端操作入口

### 4.1 发送中心 — 积分排行 Tab
- 配置栏：Top N、排序指标、话术风格
- CRM 群列表：勾选、绑定、批量绑定
- **「生成排行消息」**：批量生成各群排行 → 加入发送队列
- **「跨群总排行」**：生成全局 PK 消息 → 加入发送队列
- 生成结果：耗时统计、跳过原因、慢群提示

### 4.2 运营计划 — 积分运营模式
- 创建计划时选择 `积分运营` 模式
- 自动生成 6 个阶段（含预设节点）
- 每个阶段可编辑：标题、触发规则（类型+时间）、节点内容
- 发布时配置：目标群、起止日期、是否需要审批

## 五、性能保障

| 优化项 | 措施 |
|--------|------|
| 成员加载 | 单 SQL 批量查询所有群成员 |
| 洞察查询 | 所有群的 point_logs 合并为 1 次查询（原来 N 次串行） |
| 结果缓存 | 120 秒 TTL 内存缓存，避免重复 CRM 查询 |
| SQL 优化 | 去除 ORDER BY、分批 500 条防 SQL 过长 |
| 降级机制 | 洞察查询失败时自动降级为纯排行（无话术） |
| 诊断日志 | 分阶段计时（群名查询/成员加载/洞察分析/消息生成/绑定检查） |

## 六、当前限制与待优化

| 项 | 现状 | 建议 |
|----|------|------|
| 1v1 发送 | 生成模板供运营确认，未自动化发送 | 接入企微私发 API 实现一键发送 |
