# 飞书文档集中管理 UI 优化开发规划

## 1. 概述
本规划基于最新输出的两版 UI 设计稿（飞书文档集中管理主台、项目详情看板），旨在指导前端团队完成飞书文档模块的视觉重构与交互升级。本次优化的核心目标是**提升业务处理效率、强化视觉引导、降低信息寻找的时间成本**。

## 2. 核心架构拆解

系统将采用成熟的**左右栏（或左中右布局）**，主内容区采用柔和的背景色，卡片式容器包裹各功能模块，营造现代 SaaS 质感。

### 2.1 页面一：飞书文档集中管理（主台）
作为系统的核心入口，承担大盘概览与快速路由的功能。
*   **顶部指标区 (Stat Cards)**：横向排列的四大数据统计卡片（待处理、今日必看、当前在用、最近更新）。
*   **高优关注区 (Priority Section)**：以突出的大卡片形式展示“今日必看·优先处理”文档，包含“为什么要做”的直观引导标签。
*   **文档列表区 (Data Table with Tabs)**：多 Tab 切换（全部、当前在用、待处理等），集成复合筛选器（项目、阶段、分类、搜索），结合带有状态 Tag 的数据表格。
*   **右侧快捷侧边栏 (Right Sidebar)**：“我负责的项目”列表，提供各项目的快速导航。

### 2.2 页面二：项目详情工作台
针对具体项目进行精细化文档管理。
*   **项目 Header**：展示项目基本信息、执行状态、包含文档总数，以及主要操作按钮（编辑工作台、登记链接）。
*   **当前优先文档 (Featured Docs)**：横向两列卡片布局，重点突出当前急需处理或必读的文档及推荐理由。
*   **分类文档列表 (Categorized Lists)**：按照“执行中”、“方案设计”等阶段分组展示的长列表。
*   **右侧辅助信息面板**：
    *   快捷选项（标签云）
    *   最近更新（时间轴/列表形态）
    *   项目提醒（消息通知卡片）

## 3. 前端组件化方案 (Component UI)

为了保证代码的复用性与后期维护，需将 UI 拆解为以下基础与复合组件：

### 3.1 基础 UI 组件 (Base UI)
*   `StatusBadge.vue`: 状态标签组件，支持不同 type (success, warning, info, default)，对应不同颜色组合（如`待处理`为浅红底红字，`当前在用`为浅绿底绿字）。
*   `PriorityTag.vue`: 优先级标签组件，带图标（如向上的红色箭头代表高优先级，向下绿色箭头代表低优先级）。
*   `ActionCard.vue`: 标准信息卡片外壳，包含统一的阴影、圆角（`rounded-lg`）、悬浮交互（hover:shadow-md）。

### 3.2 业务组件 (Business Components)
*   `TopStatCard.vue`: 顶部数据统计卡片（包含图标、标题、数值描述）。
*   `FeaturedDocCard.vue`: 重点文档展示卡片，支持传入“推荐理由”、“关联项目”、“状态标签”等 props。
*   `DocFilterBar.vue`: 文档列表上方的复合筛选栏（支持下拉选择、排序、搜索）。
*   `ProjectTimeline.vue`: 右侧的最近更新时间轴组件。

## 4. 状态管理与数据流转 (State Management)

建议使用 `Pinia` 管理跨组件状态，特别是以下数据需要在多组件间共享：
1.  **看板统计数据**：顶部指标卡片的数据，需与列表数据保持一致性。
2.  **筛选与排序状态**：全局工作台的查询条件，建议和 URL query 绑定，确保刷新不丢失查询状态。
3.  **右侧边栏数据**：“我负责的项目”及“项目提醒”数据可通过统一的接口异步加载，并缓存在全局 store 中以防重复渲染。

## 5. CSS / 样式规范

当前项目采用 Vue3 + TypeScript 构建，样式层建议实施以下规范：
*   **色彩体系规范化**：
    *   主色调 (Primary)：飞书绿/系统主色系（如：`#00B96B` 或类似）。
    *   背景色 (Background)：页面统一底色采用非常浅的灰白色（如：`#F5F7F9` 或 `#F8F9FA`），以凸显纯白背景的 Card 容器。
    *   强调色 (Accent)：高优标签（Red）、进行中标签（Blue/Green）、警告栏（Orange/Yellow）需建立局部 CSS Variables 或 Tailwind Preset。
*   **间距与排版 (Spacing & Typography)**：
    *   严格执行 4px / 8px / 16px / 24px 网格节奏。
    *   卡片内部 padding 建议采用 `p-5` 或 `p-6`，保持呼吸感。
    *   列表行文字字号主次分明（主标题 14px 加粗，辅助信息 12px 浅灰色 `text-gray-500`）。

## 6. 落地实施规划 (Implementation Phases)

### Phase 1: 骨架与路由重构
*   搭建新的左右/左中右容器 Layout 组件。
*   创建“飞书集中管理”和“单项目详情”两个空白视图。

### Phase 2: 核心组件开发
*   封装并实现基础组件（`StatusBadge`, `PriorityTag`, `ActionCard`）。
*   利用 Mock 数据开发核心复合组件（`FeaturedDocCard`, 统计区块）。

### Phase 3: 业务数据对接及细化
*   对接后端 API 真实数据，完成文档列表的多 Tab 切换、筛选联调。
*   实现项目侧边栏最近更新记录和项目成员信息展示。

### Phase 4: UI 动效与交互优化
*   添加卡片 Hover 过滑动画。
*   添加页面路由切换及区块加载时的骨架屏 (Skeleton Loading)。
*   优化不同分辨率下的响应式适配（折叠侧边栏、窄屏卡片响应式）。
