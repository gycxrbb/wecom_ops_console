# CRM 群积分排行“上周积分明细”功能设计与实现方案

## 一、 背景与需求
目前系统的“群积分排行”推文仅展示成员名及其对应的核心积分（已统一调整为“当前积分”）。
为了更好地激励社群用户，现需在原有的排行消息中，为**上榜用户追加上一周的积分获取明细**。该推文常用于每周一发送，向群成员展示其排名的同时，回顾上周不同维度的活跃与得分情况。

### 核心规范
1. **排行显示字段**：统一称呼为“当前积分”，绝对不能叫“可用积分”。
2. **明细分类**：对 `point_logs` 原始繁杂的数据进行按组收编，最终展示结果仅包含 **三大类**：
   - **APP内** (如打卡、阻碍填报、课程学习、体重上传、餐食上传等)
   - **社群活跃** (如社群发言、互动等)
   - **直播积分** (如观看直播、打卡直播等)

---

## 二、 数据字典与映射规则
基于对 CRM 数据库 `point_logs` 表字段 `category`（主分类） 和 `des`（详情描述）的调研，制定如下底层数据到三大展现维度的映射关系：

### 1. APP内 (APP In-app Points)
覆盖了所有基于产品功能本身产生的积分变动。
**数据特征：**
- `category = 1` 并且 `des` 中包含以下关键字：
  - `habit_checkin` (习惯打卡)
  - `issue` (阻碍填报)
  - `courseId` (课程学习)
  - `weight_checkin` (体重打卡)
- `category = 3` 并且 `des` 包含 `meal_upload` (餐食上传)

### 2. 社群活跃 (Community Activity Points)
覆盖了企业微信群、社群内部发言和活动的积分。
**数据特征：**
- `category = 5` (或 `des` 包含 `community_interaction`)

### 3. 直播积分 (Live Stream Points)
涵盖参与直播的奖励表现。
**数据特征：**
- `category = 8` (或 `des` 包含 `live_participation`)

> **[注意]** `category = 6` (系统调整/人工加减分) 等目前不在三大类之列。如遇到则不计入这三大展示维度，以免干扰 C 端用户的感知。针对流水的增减类型，按 `type = 0` (收入) 为加分，`type = 1` (支出) 为减分 进行净值汇总。

## 三、 展示排版示例

当勾选开启并生成排行消息时，在成员排行的主列表下方追加 `[上周明细: ...]`：

```text
📊 VIP体验群-01
群总分: 15400（本周+300，本月+1200）

🏆 成员排行榜:
🥇 1. 张三先生 (当前积分: 1200)
    [上周明细: APP内+30, 社群活跃+15, 直播积分+5]
🥈 2. 李四小姐 (当前积分: 900)
    [上周明细: APP内+10]
🥉 3. 王五五 (当前积分: 850)
    [上周明细: 社群活跃+5, 直播积分+15]
    
💡 亮点:
• 张三先生上周APP内打卡非常活跃！
...
```
*(仅当该成员在大类中分数 `> 0` 时，才将该大类的明细打印出来，若上周全无动作则可不展示 `[...]` 行或展示“无”)*

## 四、 技术实施路径 (Implementation Plan)

### 阶段 1: 指标数据层增强 (`app/services/crm_points_metrics.py`)
需要增加一个新的查询函数（例：`fetch_customer_last_week_breakdown_bulk`），专门用于提取指定成员在上周时间范围内的积分明细。
- **SQL 处理策略**：通过单一查询使用 `SUM(CASE WHEN ...)` 完成这三大维度的直接归类和净值聚合，从而避免 Python 内存里的大循环计算，降低时间复杂度（参考原先双维度周月聚合的做法）。

### 阶段 2: 积分排行渲染层对接 (`app/services/crm_points_ranking.py`)
- 在原型函数的参数列表中新增控制配置 `include_last_week_breakdown: bool = False`。
- 如果参数为 `True`，取出 `ranked_members` 截取出来的候选人 ID（Top N），并调用阶段 1 的归类函数。
- 在 `_format_rank_line` 文本拼接环节，向排行榜成员文字的下一行加入该成员的 `[上周明细: ...]` 字符串。

### 阶段 3: 接口与前端参数打通 
1. **API层 (`app/routers/api_crm_points.py`)**: 将 `include_last_week_breakdown` 加到 `RankingPreviewReq` 模型中。
2. **前端与运营页面 (`PointsRankingTab.vue` 等)**: 在操作面板的复选框中新增“展示上周积分明细(仅微信/Markdown可见)”。勾选后触发后端计算。针对后台发推任务也补齐对该参数的持久化和传递。
