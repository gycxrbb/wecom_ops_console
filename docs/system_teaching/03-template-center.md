# 模板中心与运营编排

模板中心同时承担两套能力：

1. 模板库
2. 运营编排

## 什么时候用模板库

当你只是想复用“单条消息内容”时，用模板库更合适。

模板库常见动作：

- 新增模板
- 预览
- 编辑
- 复制
- 删除
- 基于系统模板创建

## 什么时候用运营编排

当你要做“多天、多节点、分阶段”的运营内容时，用运营编排。

例如：

- 7 天打卡计划
- 每周社群运营节奏
- 积分运营活动

## 运营编排工作台结构

### 左侧

管理主题和 Day / Stage：

- 选择主题
- 重命名主题
- 导出 JSON / Excel
- 删除主题
- 只看未完成
- 回到待完善
- 新增 Day / Stage

### 中间

管理当前 Day / Stage：

- 修改标题
- 修改阶段目标 / 运营焦点
- 保存
- 重置
- 新增节点
- 删除节点

### 右侧

编辑当前节点内容：

- 上一个 / 下一个
- 复制节点
- 同步同类节点到其他天
- 从模板库套用
- 选择消息类型
- 编辑发送内容
- 保存
- 保存并下一条

## 推荐使用顺序

1. 新建运营主题
2. 新增 Day / Stage
3. 新增节点
4. 给每个节点配置消息类型和内容
5. 保存
6. 预览和检查
7. 点击“发送到群”发布

## 积分运营模式

当你创建积分运营计划时，会出现积分运营专属配置，例如：

- 每日触发
- 每周触发
- 倒计时范围
- 倒数第 N 天
- 手动触发

这类计划更适合积分榜、冲刺提醒、阶段复盘等场景。
