# 话术分类管理优化方案

## 现状分析

### 当前架构

- **两级分类**：`speech_categories` 表支持自引用 `parent_id`，`level=1` 为大类（如"健康管理类"），`level=2` 为子类（如"饮食管理"）
- **分类分配**：`speech_templates.category_id` 关联到 L2 子类
- **前端渲染**：L1 > L2 > 场景列表，`sidebarTree` 计算属性按 `category_l1/category_l2` 分组

### "未分类"出现的原因

`uncategorizedScenes` 是前端计算属性，过滤出 `category_l1` 或 `category_l2` 为空的场景。这些场景的 `category_id` 为 NULL，可能是因为：

1. `SCENE_CATEGORY_MAP` 没有覆盖所有 `scene_key`（如 `rag_import` 不在映射中）
2. 手动/CSV 导入的模板未指定 `category_id`

### 现有后端 API

| 端点 | 功能 |
|------|------|
| `GET /categories` | 返回 L1+L2 树（含子类模板计数） |
| `POST /categories` | 创建分类（自动判断 L1/L2） |
| `PUT /categories/{id}` | 更新分类名称/排序 |
| `DELETE /categories/{id}` | 软删除（有子类/模板时拒绝） |
| `POST /assign-category` | 批量分配场景到分类 |

### 问题

1. 前端没有使用 `/categories` API 做分类管理，分类完全由后端 seed 和 `/scenes` 接口间接驱动
2. 无法在界面上重命名、删除、移动分类
3. "未分类"场景无入口归类

---

## 方案设计

### 目标

让分类树像文件夹一样可编辑：重命名、新建、删除、移动（拖拽排序 + 跨分类移动场景）。

### 改动范围

| 文件 | 改动 |
|------|------|
| `app/routers/api_speech_templates.py` | 新增移动分类 `PUT /categories/{id}/move` |
| `frontend/src/views/SpeechTemplates.vue` | 侧边栏分类节点加编辑/删除/新建按钮，支持内联重命名 |

---

## 详细设计

### 1. 消灭"未分类" — 补全 SCENE_CATEGORY_MAP

**根因修复**：在 `app/services/crm_speech_templates.py` 的 `SCENE_CATEGORY_MAP` 中补上缺失的场景映射。

当前缺失的场景（在 `SCENE_LABELS` 中存在但不在 `SCENE_CATEGORY_MAP` 中）：
- `rag_import`（语料导入）→ 归入"内容管理"
- 其他新增的自定义场景

同时在前端：如果仍有未分类场景，在"未分类"区域显示"归类"按钮，点击弹出分类选择器，调用 `POST /assign-category` 完成分配。

### 2. 侧边栏分类节点交互

#### L1 节点（大类）

悬停时右侧显示操作按钮组：

```
[📁 健康管理类]        [+ ✏️ 🗑]
  ├ 饮食管理 (3)
  ├ 运动管理 (2)
  └ ...
```

- **+** 新建子类：弹出行内输入框，输入名称后调 `POST /categories`
- **✏️** 重命名：点击后文字变为可编辑 input，回车或失焦调 `PUT /categories/{id}`
- **🗑** 删除：二次确认后调 `DELETE /categories/{id}`（有子类时提示清空子类后才能删）

#### L2 节点（子类）

同理，悬停时显示操作按钮：

```
  ├ [饮食管理] (3)    [✏️ 🗑 ↔]
```

- **✏️** 重命名
- **🗑** 删除（有模板时提示先移走模板）
- **↔** 移动到其他大类：弹出选择器，调 `PUT /categories/{id}/move`

#### 场景节点

悬停时显示 **归类** 按钮，可将其移到其他子类下：

```
    ├ 晒餐打卡 meal_checkin  [📋]
```

### 3. 后端新增 API

#### `PUT /categories/{id}/move` — 移动子类到另一个大类

```python
class CategoryMoveReq(BaseModel):
    new_parent_id: int

@router.put('/categories/{category_id}/move')
def move_category(category_id: int, req: CategoryMoveReq, ...):
    """Move a L2 category to a different L1 parent."""
    row = db.query(SpeechCategory).get(category_id)
    # 校验: 目标 parent 必须是 L1
    new_parent = db.query(SpeechCategory).get(req.new_parent_id)
    if not new_parent or new_parent.level != 1:
        raise HTTPException(400, '目标必须是一级分类')
    row.parent_id = req.new_parent_id
    db.commit()
    return {'ok': True}
```

#### 已有但前端未用的 API

- `PUT /categories/{id}` — 重命名（已实现）
- `DELETE /categories/{id}` — 删除（已实现）
- `POST /categories` — 新建（已实现）
- `POST /assign-category` — 分配场景到分类（已实现）

### 4. 前端改动

#### 侧边栏数据源改造

当前 `sidebarTree` 从 `/scenes` 返回的 `category_l1/category_l2` 字段构建。改为：

1. 页面加载时同时请求 `/categories` 获取完整分类树（含 id、sort_order、template_count）
2. `/scenes` 返回的场景通过 `category_id` 挂载到分类树上
3. 分类操作后刷新 `/categories`，场景操作后刷新 `/scenes`

#### 侧边栏节点增加操作交互

```vue
<!-- L1 节点 -->
<div class="speech-cat-l1" @mouseenter="showL1Actions(l1.id)" @mouseleave="hideL1Actions">
  <span ...>{{ l1.name }}</span>
  <div class="speech-cat-actions" v-if="activeL1Actions === l1.id">
    <el-button size="small" link @click.stop="openCreateL2(l1)"><Plus /></el-button>
    <el-button size="small" link @click.stop="startRename(l1)"><Edit /></el-button>
    <el-button size="small" link @click.stop="confirmDeleteCategory(l1)"><Delete /></el-button>
  </div>
</div>

<!-- L2 节点 -->
<div class="speech-cat-l2" @mouseenter="showL2Actions(l2.id)" @mouseleave="hideL2Actions">
  <span ...>{{ l2.name }}</span>
  <span class="speech-cat-count">{{ l2.template_count }}</span>
  <div class="speech-cat-actions" v-if="activeL2Actions === l2.id">
    <el-button size="small" link @click.stop="startRename(l2)"><Edit /></el-button>

  </div>
</div>
```

#### 内联重命名

点击编辑按钮后，节点名变为 `<el-input>`，回车或失焦保存：

```vue

  v-model="renamingName"
  size="small"
  @keyup.enter="saveRename(l1)"
```

#### 移动分类对话框

选择目标 L1 大类，确认后调 `PUT /categories/{id}/move`：

```vue

  </el-select>
</el-dialog>
```

### 5. 新增 L1 大类入口

侧边栏底部加"新建大类"按钮：

## 注意事项

1. **软删除安全**：`DELETE /categories/{id}` 已实现软删除（`deleted_at`），有子类/模板时拒绝
2. **缓存刷新**：分类操作后需调用 `invalidate_cache()` 刷新话术缓存
3. **sort_order 排序**：新建分类时自动取同层级最大 sort_order + 1
4. **向后兼容**：不改变 `/scenes` API 的返回结构，前端改造后可同时兼容新旧数据格式
